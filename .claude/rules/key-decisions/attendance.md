## Attendance â€” daily writer + school calendar

### KD #47

Attendance is sole writer of daily attendance. Markbook/Records/Parent are read-only consumers. Audit prefix `attendance.*`. `16-attendance-module.md`.

### KD #50

School calendar has five day-types (`school_day`, `public_holiday`, `school_holiday`, `hbl`, `no_class`) per `school_calendar.day_type` (migration 019). `school_day` + `hbl` are encodable (attendance grid accepts writes). The other three reject writes (409 on `/api/attendance/daily`). `is_holiday` is legacy, derived via BEFORE trigger from `day_type`. "Important dates / special events" stay on `calendar_events` as an overlay layer, not a day-type. **Audience precedence rule** (migration 037): both tables carry `audience IN ('all','primary','secondary')`; on read, an audience-specific row overrides the matching `'all'` row for the same date when the section's level type matches. Attendance writer at `app/api/attendance/daily/route.ts` resolves the section's level via `lib/sis/levels.ts::levelTypeForAudienceLookup`; preschool sections (YS-L/J/S) fall back to `audience='all'`. Unique key on `school_calendar` widened to `(term_id, audience, date)` so primary + secondary can each hold a row for the same date. `getEncodableDatesForTerm` + `isHoliday` apply the same precedence.

### KD #55

`/attendance` route split. `/attendance` is the analytics dashboard (registrar+ only); `/attendance/sections` is the section picker (teachers land here); `/attendance/[sectionId]` remains the daily writer. Teachers visiting `/attendance` redirect to `/attendance/sections`. Sidebar isActive: exact-match `/attendance`, prefix-match `/attendance/sections`.

### KD #76

Calendar audience scope + typed event categories (migration 037). `school_calendar` and `calendar_events` both carry `audience IN ('all','primary','secondary')`; `calendar_events` also carries `category IN (term_exam | term_break | start_of_term | parents_dialogue | subject_week | school_event | pfe | ptc | other)` + `tentative bool`. KD #50's 5 day-types stay the attendance gate â€” new event types live on the overlay layer, NOT as new day-types (a term-exam day is still a school day with attendance taken). UI at `/sis/calendar` exposes an `?audience=all|primary|secondary` filter tab; cell click-cycle and event create write to the active filter. Audience precedence rule: an audience-specific row beats the matching `'all'` row for the same date (KD #50 documents the read-side rule). Carry-forward stays manual: `components/attendance/copy-from-prior-ay-dialog.tsx` (renamed from copy-holidays-dialog) handles both day-type overrides + events with year-shift; default `markTentative=true` flips every copied row to `tentative=true` so the registrar reviews each before locking. Preschool (YS-L/J/S) deferred â€” preschool sections always read `audience='all'`; future scope can extend the enum with one CHECK update + dropdown additions.

### KD #94

Vacation-leave subtype + per-term quotas + configurable defaults (migration 048). Verified against HFSE's T1 attendance workbook: the registrar's spreadsheet header carries two policy labels ("4 Vacation Leaves: 1 per term" + "5 days (per year) Urgent/Compassionate Leave"), but only one subtype existed in the schema (`ex_reason âˆˆ mc | compassionate | school_activity`); vacation was recorded as a free-text Excel cell comment ("1 VL approved", "took vacation leave for outing"). Now structured: **`attendance_daily.ex_reason` check widened** to include `'vacation'`; **`students.vacation_leave_allowance_per_term smallint`** added as a nullable per-student override (NULL = use school default â€” deliberately cleaner than `urgent_compassionate_allowance`'s `NOT NULL DEFAULT 5`, which is left as-is to avoid a breaking change); **`school_config` gains `default_compassionate_allowance_per_year smallint default 5`** + **`default_vl_allowance_per_term smallint default 1`** â€” these become the canonical school-wide defaults the new `<VacationAllowanceInline>` reads when a student row is NULL. Partial index `attendance_daily_vacation_idx (section_student_id, term_id) where ex_reason='vacation'` for per-term quota counters. **Read path**: `getVacationLeaveUsage` / `getVacationLeaveUsageForSection` in `lib/attendance/queries.ts` mirror the compassionate helpers but scope on `term_id` (no AY-wide carry-forward). **Drill**: new `'vacation-leave-quota'` target on `AttendanceDrillTarget`, `'vacation-leave'` row-kind, `VacationLeaveUsageRow` row shape, `rollupVacationLeave` rollup. `lib/attendance/drill.ts` stays client-bundle-safe by accepting `defaultVlAllowance` as a parameter (caller fetches from `school_config` and passes in) instead of importing the server-only school-config module at top-level â€” important: when a server-only module reaches a client component via a value import in any transitively-imported file, Next 16 errors with `'server-only' cannot be imported from a Client Component module`. Drill API route forwards `?termId=` so per-term filter ships server-side. **UI**: wide-grid gains `EX:vacation` as the 4th EX subtype (letter mnemonic `EV` derives via existing `slice(3,4)` logic â€” no extra branch); soft warning toast fires from `writeCell` when a 2nd-VL-in-term would exceed `vlAllowance` (write still proceeds â€” registrar can grant exception). New `<VacationLeaveQuotaCard>` on the attendance dashboard with `Umbrella` icon + `brand-sky â†’ brand-indigo` gradient distinguishes it from the compassionate card. Per-student `/attendance/students/[studentNumber]` page mounts both cards in a `md:grid-cols-2`. New `<VacationAllowanceInline>` mirrors `<CompassionateAllowanceInline>` but treats NULL as the canonical "use school default" state â€” Reset writes NULL back so future school-default changes propagate. New `PATCH /api/sis/students/[enroleeNumber]/vl-allowance` route + `'sis.vl_allowance.update'` audit action. SIS Admin school-config form gains an "Attendance quotas" section group; the route's `requireRole` widens from `['superadmin']` to `['superadmin', 'school_admin']` per KD #39. **Out of scope**: bulk Excel import doesn't yet parse cell comments â†’ VL subtype (future enhancement); priority-panel VL chip deferred (the dashboard card is the primary signal).

### KD #113

Late-enrollee attendance proration (migration 068). `recompute_attendance_rollup(p_term_id, p_section_student_id)` now fetches `section_students.enrollment_date` and filters `attendance_daily` entries to `date >= enrollment_date` before counting school_days / present / absent. For active students (`enrollment_date IS NULL`) behaviour is unchanged. For late enrollees joining in T3, T1 and T2 rollups produce 0/0/0 stats instead of counting any pre-enrollment entries that exist due to backfill. **Wide-grid**: `app/(attendance)/attendance/[sectionId]/page.tsx` now selects `enrollment_date` from `section_students` and passes it through `WideGridEnrolment`; cells whose calendar date is before the student's `enrollment_date` render as non-interactive dimmed spans (`bg-muted/40 opacity-40`) with a "Before enrolment date" tooltip â€” existing back-dated entries still display dimmed rather than being silently hidden. Extends KD #68 (late-enrollee term detection).

### KD #98

School-holiday HBL overlay (migration 051). `school_calendar.hbl_overlay boolean NOT NULL DEFAULT false` makes a `day_type='school_holiday'` row simultaneously encodable â€” teachers submit attendance as on a standalone HBL day. `isEncodableDayType` in `lib/schemas/attendance.ts` + the write-gate in `app/api/attendance/daily/route.ts` treat `school_holiday + hbl_overlay=true` as encodable. The `is_holiday` BEFORE trigger is NOT changed â€” overlaid rows remain `is_holiday=true`. Extends KD #50's model: 5 day-types stay the canonical gate; HBL overlay is a per-row modifier only meaningful when `day_type='school_holiday'`. Matches HFSE's published AY2026 calendar where marking-day and awards-deliberation day carry both "School Holiday" and HBL labels on the same cell.
