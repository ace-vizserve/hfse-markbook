### KD #67
Atomic mid-year section transfer. Reassigning an enrolled student to a new section within the same level happens through the dedicated `POST /api/sis/students/[enroleeNumber]/transfer-section` route, NOT the existing class-stage PATCH. Helper `lib/sis/section-transfer.ts::transferStudentSection` runs an atomic withdraw-old-row + insert-new-row pair against `section_students` (per Hard Rule #6: append-only) plus an admissions-side `classSection`/`classLevel`/`classStatus='Finished'` update on the same call. The class-stage PATCH rejects post-Enrolled `classSection` mutations with a 422 pointing to the new endpoint â€” closes the silent dual-section bug where the per-row sync's narrow scope left the student active in BOTH sections. Pre-flight rejections: cross-level transfer (same-level only), target at 50-cap (Hard Rule #5), same-section, missing studentNumber. Term-of-transfer captured in audit context via shared `lib/sis/terms.ts::getTermForDate`. UI: per-row "Move" action on the SIS Admin section roster table at `/sis/sections/[id]`. Records detail page renders a "Section transfers" history card from `lib/sis/section-history.ts` (audit-log-derived, cross-AY).
