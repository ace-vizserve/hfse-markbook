## Dashboards — framework, drills, compare-mode

### KD #45
One consolidated dashboard per module; no cross-module data leaks. `/records` absorbs admissions analytics because Records *is* the operational admissions surface.

### KD #46
Each data dashboard owns `lib/<module>/dashboard.ts` (SIS also owns `health.ts`). Cache-wrapper pattern: hoist `load*Uncached`, compose `unstable_cache` per-call for dynamic tags. Consistent hero + trust-strip shape.

### KD #54
Dashboard framework. Module dashboards compose from `lib/dashboard/*` + `components/dashboard/*` (DashboardHero, ComparisonToolbar, MetricCard, InsightsPanel, ActionList + recharts wrappers). Per-module `lib/<module>/dashboard.ts` adds `*Range` siblings to AY-scoped helpers (KD #46 cache-wrapper pattern preserved). URL-param contract: `?ay=&from=&to=&cmpFrom=&cmpTo=&status=&level=` — no dropdown UI for secondary filters. "Target" = auto-computed prior-period comparison; no stored `kpi_targets`. Chart budget ≤ 8/screen. **Gotcha:** `lib/dashboard/windows.ts` uses `createServiceClient()` — cookie-scoped clients are forbidden inside `unstable_cache` in Next 16.

### KD #56
Drill-down framework. Every aggregating dashboard surface is a clickable drill into the underlying rows. Per-module shape: `lib/<module>/drill.ts` (row shape(s) + `buildDrillRows` + `applyTargetFilter` + `defaultColumnsForTarget` + `drillHeaderForTarget`); `app/api/<module>/drill/[target]/route.ts` (unified GET, JSON+CSV with UTF-8 BOM); `components/<module>/drills/{<module>-drill-sheet,chart-drill-cards}.tsx` (per-target `'use client'` wrappers — Server Components cannot pass functions as children to Client Components in Next 16). Shared infra: `components/dashboard/drill-down-sheet.tsx` + `MetricCard.drillSheet` slot (signature `() => ReactNode` per Phase D.ii — render-prop defers JSX construction so the heavy `@tanstack/react-table` subtree only constructs when the user opens a drill). Pre-fetch returns rolled-up shapes only; raw row arrays lazy-fetch on drill open with `DrillSheetSkeleton`. **Drill always reflects the page-level date range picker** — the prior in-drill scope toggle was removed in Sprint 36; the page-level `<DateRangePicker>` is the single source of truth, with `defaultPreset='thisMonth'` for flexible modules and `'thisTerm'` for academic modules per KD #79. Auth: per-module `requireRole(...)` + teacher row-scoping where applicable. Cache: per-call `unstable_cache` with module-scoped tag (`${module}-drill:${ayCode}`), 60s revalidate. Shared `lib/auth/teacher-emails.ts::getTeacherEmailMap()` replaces ad-hoc `auth.admin.listUsers` calls. Each drill row's primary identifier (studentName / fullName / sectionName / enroleeFullName) is an inline `next/link` to the canonical detail surface — see KD #81 for the per-module destination map.

### KD #57
Layout archetypes. Every module dashboard is one of three: **operational** (top-of-fold = `PriorityPanel`); **analytical** (top-of-fold = F-pattern KPI grid + `InsightsPanel`); **hub** (top-of-fold = admin nav cards, KPIs only via opt-in `?view=audit`). Shared primitive `components/dashboard/priority-panel.tsx` + `lib/dashboard/priority.ts` (`PriorityPayload` type). Per-module priority computers live next to existing `dashboard.ts` loaders. Role-aware composition: where one URL serves both teachers and registrars (Markbook + Evaluation), the page RSC branches on `sessionUser.role` at SSR — teachers stay at the same URL, never a per-role route. Module assignments in `20-dashboards.md` § Layout archetypes. Page-level role differentiation across other modules in KD #74.

### KD #74
Page-level role differentiation closes the read-only oversight gap. Three module dashboards previously treated all allowed roles as the same viewer; this adds an `isOperational` (or `isOfficer`) flag at the page RSC and gates operational top-of-fold tiles behind it. **P-Files** (`/p-files`, `/p-files/[enroleeNumber]`): `isOfficer = role === 'p-file' || role === 'superadmin'` gates `<PriorityPanel>` + `<DocumentChaseQueueStrip>` + bulk-notify; school_admin sees hero copy reframed as "Read-only oversight" + analytical KPIs only. **Records** (`/records`): `isOperational = role === 'registrar'` gates chase strip + "Documents to collect" `ActionList` + `<ClassAssignmentReadinessCard>`. **Admissions** (`/admissions`): `isOperational = role === 'admissions' || role === 'registrar'` gates `<NewApplicationsPriority>` + chase strip + chase priority + chaseInsights; oversight roles keep KPIs + drill cards. **SIS Admin** (`/sis`): hero copy branches by access tier; Access + System sections (Approvers / School Config / Users / Settings) accessible to school_admin + superadmin per Sprint 33 (KD #39). **Sidebar prose** (`lib/auth/roles.ts::PFILES_NAV`): "Expiring soon" quicklinks gated to `['p-file', 'superadmin']` (officer-only); "Document validation" available to `school_admin` too. KD #57 stays the umbrella.

### KD #78
Compare-mode framework (Sprint 35). Per-module `/<module>/compare` sub-routes deliver multi-AY × multi-term (academic) or × multi-month (flexible) side-by-side analytics — first-class peer of the operational dashboard, not a comparison overlay on it. URL contract: `?ays=AY9998,AY9999&terms=T1,T2` (academic) or `&months=2026-04,2026-05` (flexible). Shared infra: `lib/dashboard/compare.ts` (`CompareInput` discriminated union over `kind: 'term' | 'month'`, `parseCompareParams`, `buildCompareCells` cross-joins ays × terms/months with one cross-AY terms query); per-module `lib/<module>/compare.ts` fans out via `Promise.all` over the existing `getXxxKpisRange` per cell — zero new DB query patterns, per-cell `unstable_cache` shares slots with operational view. UI primitives: `<CompareToolbar>` (multi-AY + multi-term/month cmdk picker), `<CompareGrid>` (rows = metric, cols = cell, mint/destructive highlight on extremes via `highlightExtremes` flag), `<MultiSeriesTrendChart>` + `<MultiSeriesComparisonBarChart>` (parallel siblings to existing 2-series charts; legacy charts untouched). Module classification: term-scoped (Markbook/Attendance/Evaluation) take `kind="term"` + role gate `[registrar, school_admin, superadmin]` per KD #74; flexible (Admissions/Records/P-Files) take `kind="month"` + their respective operator roles. Operational `cmpFrom/cmpTo` URL params unchanged — compare-mode is a parallel system. Two test AYs (`AY9999` current calendar year + `AY9998` year-shifted prior year) provisioned together via `ensurePriorTestAy` + `seedPriorYearTestAy`.

### KD #79
Picker preset split — term-scoped vs flexible (Sprint 35). Module dashboards pass one of two preset shortlists to `<ComparisonToolbar presets={...}>`, which forwards to the underlying `<DateRangePicker>`. **Term-scoped** (Markbook/Attendance/Evaluation): `TERM_SCOPED_PRESETS = ['thisTerm', 't1', 't2', 't3', 't4', 'thisAY', 'custom']`. **Flexible** (Admissions/Records/P-Files): `FLEXIBLE_PRESETS = ['thisMonth', 'lastMonth', 'lastWeek', 'last15d', 'thisAY', 'custom']`. New presets in `lib/dashboard/range.ts::Preset`: `t1/t2/t3/t4` (resolve via `TermWindows.byNumber: {1,2,3,4}`), `lastWeek` (previous Mon–Sun), `last15d` (rolling today−15..today−1), `thisMonth` (1st of current month → today MTD), `lastMonth` (previous full calendar month). `resolveRange` accepts `{ defaultPreset?: Preset }` so flexible pages default to `'thisMonth'` instead of the legacy `thisTerm` cascade. `detectPreset` accepts the picker's `presets` shortlist so the active-preset chip never lights up on a preset the user can't see. `windows.ts` cross-AY active-term fallback: when no term in current AY contains today and no `is_current` flag is set, `thisTerm` falls back to most-recently-finished term in any prior AY; `activeTermFallback: boolean` returned for the page-level "showing previous term" banner. **Custom-mode fix (Sprint 36)**: `<DateRangePicker>` no longer auto-commits when the user picks a second date — the popover requires an explicit Apply click, so multi-month custom ranges work.

### KD #80
Drill cache invalidation contract (Sprint 36 Phase B). Every mutation route that touches a drill-cached table calls `invalidateDrillTags(module, ayCode)` from `lib/cache/invalidate-drill-tags.ts` after the DB write succeeds, before the response. Tag pattern is `${module}-drill:${ayCode}` matching the per-AY tags declared by the rolled-up loaders in `lib/<module>/drill.ts`. Cross-cutting routes (`app/api/sections/[id]/students/*`, `app/api/sis/students/[enroleeNumber]/transfer-section`, `app/api/students/sync`) use `invalidateAllOperationalDrills(ayCode)` which fans out to all 5 operational modules. After the seeder runs, `seedPopulated` ends with the same call so a freshly-seeded environment renders without waiting for the 60s `unstable_cache` TTL. Browser-side: drill API routes set `Cache-Control: private, max-age=60, stale-while-revalidate=300` on JSON responses (CSV exports unchanged). Phase A + D.i shipped alongside: chart wrappers in `components/dashboard/charts/` use `next/dynamic` (recharts loaded on demand); `<MetricCard>.drillSheet` is a render prop (`() => ReactNode`) so each card's drill JSX only constructs at click-time.

### KD #81
Drill primary-identifier linkification (Sprint 36). Every drill sheet's row identifier (`studentName` / `fullName` / `sectionName` / `enroleeFullName`) is an inline `next/link` styled `font-medium text-foreground transition-colors hover:text-primary hover:underline underline-offset-4`. Per-module destinations: Markbook entry → `/records/students/[studentNumber]` (KD #4 cross-year); Markbook sheet + change-request → `/markbook/grading?section=[sectionId]`; Attendance entry / top-absent / compassionate → `/attendance/students/[studentNumber]`; Attendance section-summary → `/attendance/[sectionId]`; Evaluation writeup → `/records/students/[studentNumber]`; Admissions → `/admissions/applications/[enroleeNumber]`; Records → `/records/students/[studentNumber]` with `/records/students/by-enrolee/[enroleeNumber]` fallback when `studentNumber` is null; P-Files → `/p-files/[enroleeNumber]` (rows pre-filtered to enrolled per KD #71); Lifecycle drill is **stage-conditional** — `applicationStatus` ∈ `Enrolled` / `Enrolled (Conditional)` (KD #59) routes to Records, otherwise Admissions. The mono ID line below the human name stays plain text. Codes (subject / term / grade-bucket) and dates are not linkified.

### KD #82
Drill scope-anchor by target (Sprint 36). `lib/admissions/drill.ts::applyScopeFilter` accepts an optional `target` parameter and picks the date column to filter by based on the target — chart and drill must share the same anchor or segment clicks open empty. **Per-target anchors:** `'enrolled'` / `'avg-time'` use `enrollmentDate` (matches `computeRangeKpis` which counts enrolments completed in range); `'time-to-enroll-bucket'` skips the date filter entirely (chart histogram is AY-wide); everything else uses `applicationDate` (the canonical "applications received in this range" anchor). `applyTargetFilter` for `'pipeline-stage'` filters on `r.pipelineStage` (derived from rightmost-set `*UpdatedDate` column matching `loadPipelineStageBreakdown`'s logic) and `'day-type'` does a label→enum reverse-lookup so donut clicks (`'School day'`) match raw DB values (`'school_day'`). Pattern: when a chart and a drill diverge on row counts, check whether they share scope anchor + segment-key vocabulary first. **Sprint 37 audit found two more cases of the same bug class — both fixed:** Markbook `term-sheet-status` drill regex matched only the compact `T1:locked` form, but `<SheetProgressChart>` emits the human label `Term 1 · Locked` → fell through to `return rows` (= every sheet); now uses dual-regex like `term-publication-status` already did. P-Files `slot-by-status` drill matched `r.status === segment` only, but the donut's `'Expired'` slice intentionally combines Expired + Missing per the chart's documented design (`slotMix.missing` lumps both); now unions both statuses on the Expired branch + explicitly matches `'On file'` (the post-normalization label, not the raw `'Valid'`). **Sprint 37 also fixed four date-picker label/scope mismatches** on the dashboards (audit via cross-module review): Records `SummaryStat` footnote read "Across all enrolled students" while the value was range-scoped (now "From end of selected range"); Markbook `lockedPct` comparison label mixed a range numerator with an AY-wide denominator (replaced with honest `"of N sheets in this AY"`); P-Files `RevisionsOverTimeChart` + `RevisionsHeatmapCard` were fixed 12-week trailing windows masquerading as range-aware (eyebrow now reads "Rolling" to make the picker-independence explicit).
