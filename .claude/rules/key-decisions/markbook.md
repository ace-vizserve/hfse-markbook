## Markbook — grading, change-requests, report cards, publishing

### KD #3

### KD #6

### KD #75

### KD #88

Grade change-request hardening (migrations 044 + 045). Three correctness + workflow + polish layers on KD #25's flow. **Migration 044**: 10 new nullable columns on `grade_change_requests` — paired primary/secondary review trail (KD #41 dual-reviewer audit) + `eligible_approver_snapshot jsonb` (write-once pool capture) + `notification_status`. Ships `apply_change_request_atomic` SECURITY DEFINER RPC that re-checks `grading_sheets.is_locked FOR UPDATE` and applies the entry patch + flips status to `'applied'` atomically — closes the silent race where a registrar's mid-apply unlock could bypass Hard Rule #5's `approval_reference`. **Migration 045**: `approved_at` (canonical, separate from primary_reviewed_at), `reminder_sent_at` (idempotency for the lazy reminder), `rejection_undone_at` + a partial index for the candidate scan. **Server enforcements**: 422 when applicant has no `studentNumber` at Enrolled-flip; canonical numeric `valuesMatch` (`'85'` matches `85.0`); same-person double-stamp guard + concurrent-approval `.eq('status', 'pending')` optimistic gate; spurious-value 422 (proposed === current); `slot_index` ceiling check against `subject_configs.{ww,pt}_max_slots`; backstop no-op audit row when applying a change-request whose value already matches stored (so the approval_reference trail is never empty). **Workflow**: 2-hour rejection undo via `action: 'undo_rejection'` (rejecting approver only); approved-but-not-applied reminder at 3 days via lazy fire from GET inbox (idempotent stamp-before-send); "Open sheet" CTA on teacher's approved row; aging chip ("approved 5 days ago", muted < 3d / amber 3–7d / destructive > 7d). **Polish**: `lib/env.ts` central env-var module with build-time `console.warn` when `NEXT_PUBLIC_SIS_URL` missing + amber `<SisUrlMissingBanner>` on `/sis/admin/settings`; `MeScopeConfig.enabled?` replaces the sentinel-string pattern in the unified `<DataTable>` shell (KD #84). New AuditActions: `grade_change_undo_rejection` + `reviewer_ordinal` in context. `<EditStageDialog>` surfaces `autoSync.reason` via `toast.warning` when class auto-assign succeeded but section-roster sync skipped.

### KD #95

Masterfile + Subject Award + Overall Academic Award (migration 049). Replicates HFSE's AY2025 Final Report Book Masterfile sheet as a registrar-facing surface at `/markbook/masterfile?level=&class=`. **Verified canonical grading spec** (user-confirmed this session, captured in `docs/context/02-grading-system.md`): Quarterly = ROUNDDOWN(transmutation, 0) integer per term; Subject Overall = ROUND((T1×0.2)+(T2×0.2)+(T3×0.2)+(T4×0.4), 2); General Average = ROUND(AVERAGE(examinable Subject Overalls), **1dp**) — fixed `lib/compute/annual.ts::computeGeneralAverage` from a 2dp → 1dp drift. Subject Award + Overall Award share the same threshold ladder and both compute via HFSE's literal IFS formula `=IFS(<88.5,"NE", <=91.4,"Bronze", <=95.4,"Silver", <=99.4,"Gold")` with the upper bound extended past 99.4 so a perfect 100 doesn't fall through. **Schema** (migration 049): flips `subjects.is_examinable=false` for the 8 letter-graded subjects (`MUSIC, ARTS, PE, HE, CL, CA, PEH, PMPD`) the original seed had wrong as `true`; adds 4 typed columns to `school_config` (`subject_award_bronze_min/silver_min/gold_min/max`) with cross-column ordering CHECK. Defaults 88.5 / 91.5 / 95.5 / 100 — editable in SIS Admin → School config (school_admin+ per KD #39). Updates `seed.sql` so fresh installs are correct. **Compute** (`lib/compute/awards.ts`): pure functions `subjectAward` + `overallAcademicAward` + `tierFor` helper; takes `AwardThresholds` + `AwardEligibility` (enrolled + hasCompleteData) so disqualifiers (withdrawn, late-enrollee with incomplete examinable cells, null overall) override the numeric tier. Module self-test on import covers all boundary cases (88.4, 88.5, 91.4, 91.5, 95.4, 95.5, 99.4, 99.5, 100). **Loader** (`lib/markbook/masterfile.ts`): per-(level × ay × class-filter) RSC loader, 60s `unstable_cache` keyed on the same set, tagged `markbook-drill:${ayCode}` so existing grade mutations invalidate it. Mirrors `build-report-card.ts` cross-section union logic for KD #67 mid-year transfers. **Grid** (`components/markbook/masterfile-grid.tsx`): native `<table>` with horizontal scroll — examinable subjects = 6 sub-columns each (T1/T2/T3/T4 + Overall + Award badge), non-examinable = 4 sub-columns (T1-T4 only, render letter from `grade_entries.letter_grade` or "—"). Sticky left column = student name (linkified to `/records/students/[studentNumber]` per KD #81). Cross-mount: sidebar entry under Markbook (next to Report Cards) + deep-link from Records sidebar under "Reports" group. ROUTE_ACCESS scopes `/markbook/masterfile` to `registrar | school_admin | superadmin` only (longer prefix wins per `isRouteAllowed`). **Out of scope (Phase 2)** pending Joann clarification: non-examinable letter entry surface (storage already exists since migration 001 — `grade_entries.letter_grade text`); Honors / High Honors / Highest Honors tier; Sec 4 Economics supplementary card; T4 report card General Average row (already computes via the 1dp fix but render-side verification deferred). The 4 Phase-2 questions: who enters non-examinable grades + from where, complete verified letter list, whether UG and E are actually used at HFSE, whether the SharePoint Consolidated Form integrates or gets replaced.

### KD #99

`sync_grading_sheets_from_config(p_config_id uuid)` RPC (migration 052b). Grading sheet `ww_totals/pt_totals/qa_total` columns are denormalized copies of subject_configs at sheet-creation time. When SIS Admin updates `ww_max_slots`, `pt_max_slots`, or `qa_max` on a `subject_config`, this RPC brings all **unlocked** sheets linked to that config into sync — locked sheets are never touched (Hard Rule #5). Resize behaviour: ww/pt arrays extended → new slots default to max score 10; truncated → trailing scores/totals dropped; `qa_total` replaced unconditionally with the new `qa_max`. Called by `PATCH /api/sis/admin/subjects/[configId]` immediately after the DB UPDATE, before the audit log write. Returns `jsonb` with `rows_synced` count for the audit trail.

### KD #100

Non-examinable `annual_letter_grade` (migration 053). Registrar-entered year-end grade for non-examinable subjects (Music, Arts, PE, HE, CL, CA, PEH, PMPD). Stored as `grade_entries.annual_letter_grade text NULL` on the T4 row. **Controlled value set** (confirmed HFSE workflow): `['Passed', 'UG', 'E', 'NA']` — exported as `ANNUAL_LETTER_VALUES` from `lib/compute/letter-grade.ts`. "Passed" is the routine year-end entry; UG/E/NA are override codes. First-time entry (null → value) saves immediately; changing an existing value requires a correction note. Hard Rule #5 does NOT apply — this is registrar metadata, not a locked score. Route: `PATCH /api/grading-sheets/[id]/entries/[entryId]/annual-letter` (registrar/school_admin/superadmin) validates against `ANNUAL_LETTER_VALUES`. UI: `<AnnualLetterInput>` inline popover in the Masterfile grid and report card T4 Final column. Publish-readiness checks that all T4 non-examinable rows have `annual_letter_grade` set or `is_na=true` (KD #28). **Distinct from `grade_entries.letter_grade`** (per-term quarterly override, KD #104).

### KD #104

Non-examinable subjects use the same WW/PT/QA score entry as examinable subjects (migration 055). The Quarterly cell displays a derived letter via `lib/compute/letter-grade.ts::numericToLetter`: A (90–100), B (85–89), C (80–84), IP (≤79). Manual override codes **UG and E** are stored in `grade_entries.letter_grade` and take precedence over the derived letter (INC/CO were removed — not in HFSE's confirmed legend). NA is the existing `is_na` checkbox. A/B/C/IP/NA are never written by the score-entry flow — they are derived at render time via `resolveNonExaminableLetter({ isNa, letterOverride, quarterly })`. Migration 055 cleared all legacy `letter_grade` values for non-examinable rows (teachers must re-enter via the score grid). `<ScoreEntryGrid letterDisplay={true}>` is the unified entry surface; the legacy `<LetterGradeGrid>` was deleted. T4 `annual_letter_grade` (KD #100) is unchanged — registrar-entered freeform, takes precedence over the derived letter on the report card Final column. `<TotalsEditor>` is now shown for all subjects (not gated on `isExaminable`). The entries PATCH route (previously short-circuiting non-examinable) now runs the full WW/PT/QA + `computeQuarterly` path for both subject types; `letter_grade` writes validate against the override allowlist only.

### KD #110

Teacher-owned SOW model (migration 061). Supersedes the admin-authoring half of KD #108. **Context**: field investigation of HFSE's actual workflow found that subject teachers — not coordinators — finalize the SOW (already a standard off-system format); coordinators do light random spot-checks. **Migration 061** reshapes `sow_class_instances`: drops `master_template_id`, `version_id`, `has_partial_rebaseline`; adds `ww_labels jsonb NOT NULL DEFAULT '[]'`, `pt_labels jsonb NOT NULL DEFAULT '[]'`, `topics jsonb NOT NULL DEFAULT '[]'`, `copied_from_section_id uuid`, `copied_at timestamptz`, `created_by uuid`. Re-adds unique key `(section_id, subject_id, term_id)`. `evaluation_checklist_items` re-scoped back to per-section: TRUNCATE (safe — all writes were 410'd since migration 058 per KD #108), drops `level_id + curriculum_track + sow_class_instance_id`, adds `section_id NOT NULL`, `sow_instance_id uuid` (provenance pointer). **SOW lifecycle**: (1) teacher authors at `/markbook/sow` — WW labels + page refs, PT labels + page refs, evaluation topics; OR imports from a peer section in the same `(level × subject)` which stamps `copied_from_section_id + copied_at`. (2) Grading sheet bulk-create calls `syncSowLabelsToSheet` after sheet creation (soft — logs if SOW missing, does not gate). (3) On the evaluation page when checklist is empty, teacher calls `POST /api/sow/[id]/sync-to-eval` to seed topics. Mid-term drift is fine — once seeded, teachers edit labels/topics on the operational surface. (4) Coordinator spot-checks at `/sis/admin/sow` (read-only — term + subject pickers, one row per section showing labels + topic count + provenance). **AY Readiness Pill** reduced from 5 to 4 steps (SOW step removed — no longer a hard gate). **New audit actions**: `sow.instance.save`, `sow.instance.import_from`, `sow.labels.synced`, `sow.topics.synced`. All four appear on SIS + Markbook audit logs; the last two also on Evaluation. **New lib**: `lib/markbook/sow.ts` (`listTeacherSowItems`, `syncSowLabelsToSheet`, `syncSowTopicsToChecklist`). **New API routes**: `GET|PUT /api/sow`, `POST /api/sow/import`, `POST /api/sow/[id]/sync-to-grading-sheet`, `POST /api/sow/[id]/sync-to-eval`. **Teacher surfaces**: `/markbook/sow` (index — list by level, status badge: empty/drafted/synced) + `/markbook/sow/[sectionId]/[subjectId]/[termId]` (editor). **Hard Rule #5 preserved**: "Sync labels to grading sheet" button disabled + 423 returned when sheet is locked. `sow_master_templates` + `sow_published_versions` fully removed by migrations 062–066 (see KD #108). Evaluation checklist CRUD un-410'd: `POST /api/evaluation/checklist-items` + `PATCH|DELETE /api/evaluation/checklist-items/[id]` — teacher-scoped, `section_id` from item. `copy-from` stays 410'd (copy-from now goes through SOW import, not directly between checklist rows).
